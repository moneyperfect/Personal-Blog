# Skills 安装清单（详细版）

> 说明：此清单基于本机目录扫描结果（`~/.gemini/skills`、`~/.codex/skills`、`~/.antigravity/skills`、`~/.gemini/antigravity/skills`）。
> 如有新增/移动，请告诉我再更新。

## 通用使用方式

- 直接点名：`使用 <skill-name> 技能 + 任务`
- 或用 `$skill-name` 触发
- 建议加上“输出要求/格式”，例如：
  - `请先提问澄清，不要直接给方案`
  - `输出为 Obsidian Markdown`
  - `给出可直接导入的 .canvas/.base`

示例：

```
$obsidian-markdown
请把下面内容整理成 Obsidian 笔记，含标题、要点、双链与 callout。
```

```
$json-canvas
把“项目拆解流程”做成 Obsidian Canvas（含节点与连线），返回 .canvas JSON。
```

## ~/.codex/skills

### json-canvas
- 主要用途：创建/编辑 Obsidian `.canvas`（节点、连线、分组、布局）
- 适用场景：流程图、思维导图、概念图、结构化可视化
- 建议触发：
  - `$json-canvas 生成 <主题> 的流程图 canvas`
- 期望输出：可直接导入的 `.canvas` JSON

#### 提示词模板
```
$json-canvas
主题：<主题>
用途：<场景>
要求：
- 节点数量：<例如 6-10>
- 必含分组：<是/否>
- 结构：<流程/层级/网状>
- 备注：请返回可直接导入的 .canvas JSON
```

### obsidian-bases
- 主要用途：创建/编辑 Obsidian `.base`（视图、过滤器、公式、汇总）
- 适用场景：笔记数据库视图（表格/卡片/列表/地图）与统计汇总
- 建议触发：
  - `$obsidian-bases 给我一个 <场景> 的 base，含过滤条件与汇总`
- 期望输出：完整 `.base` YAML

#### 提示词模板
```
$obsidian-bases
目标：<你要管理的内容>
数据来源：<文件夹/标签/属性>
视图类型：<表格/卡片/列表/地图>
字段：<需要展示的属性>
过滤：<条件>
汇总：<需要统计的字段与方式>
输出：请给完整 .base YAML
```

### obsidian-markdown
- 主要用途：生成/编辑 Obsidian Markdown（wikilink、callout、properties、embed）
- 适用场景：写笔记、整理资料、模板化输出
- 建议触发：
  - `$obsidian-markdown 把下面内容整理成笔记，并加上 properties 与双链`
- 期望输出：可直接粘贴的 Obsidian 笔记

#### 提示词模板
```
$obsidian-markdown
主题：<标题>
用途：<笔记目的>
要求：
- 需要 frontmatter 属性：<属性列表>
- 需要双链：<要链接的笔记>
- 需要 callout：<类型与内容>
- 输出结构：<如 摘要/要点/行动>
```

### think-through
- 主要用途：苏格拉底式提问，先澄清再输出
- 适用场景：需求不清晰、想被反问推动思考
- 建议触发：
  - `$think-through 先不断提问，直到需求明确再给方案`
- 期望输出：一组逐步澄清的问题 + 最终整理后的需求/方案

#### 提示词模板
```
$think-through
目标：<你想解决的问题>
规则：
- 先提问，不要给方案
- 每次只问 1-3 个关键问题
- 当信息足够时，再总结需求并给出方案
```
